Rapport de Synthèse - Stage CRBS

Évaluation de l'Effet Rescue

Ce rapport documente la méthodologie automatisée utilisée pour identifier les signatures pathologiques et évaluer l'efficacité de corrections thérapeutiques sur 7 cohortes ARN-Seq.

0. Origine des fichiers bruts : Du vivant au "Raw Count"

Avant d'arriver dans notre script Python sous forme de fichiers .txt, les données traversent le cycle du dogme central de la biologie moléculaire.

1
Transcription (ADN → ARNm) : Dans les cellules musculaires, les gènes (ADN) sont copiés en ARN messagers. C'est cette étape que nous capturons. Plus un gène est actif, plus il y a d'ARNm correspondant.
!
Note sur la Traduction (ARNm → Protéine) : En temps normal, l'ARNm sort du noyau pour être traduit en protéines par les ribosomes. Bien que nous ne mesurions pas directement les protéines (protéomique), nous utilisons l'ARN comme indicateur du flux de production cellulaire.
2
Séquençage (RNA-Seq) : Les ARNm extraits sont fragmentés et convertis en "bibliothèques" séquençables. Le séquenceur génère des millions de lectures courtes (Reads).
3
Mapping (Alignement) : Bio-informatiquement, on aligne ces Reads sur le Génome de Référence (souris). C'est l'étape de localisation : "A quel gène appartient cette séquence ?".
4
Quantification (Gene Count) : On compte le nombre de Reads alignés sur chaque gène.
Résultat : On obtient la Matrice de Comptage Brut (Raw Counts) issue du traitement bioinformatique. Chaque chiffre représente l'abondance d'un transcrit spécifique dans un échantillon donné.

1. Ce que nous cherchons à mesurer

L'objectif central est l'extraction des Gènes Différentiellement Exprimés (DEG) pour deux signatures clés :

  • La Signature Pathologique : Définie par le contraste entre le groupe malade (Disease) et le groupe sain de référence.
  • L'Effet Rescue : Définie par les gènes dont l'expression est corrigée (retour vers le niveau sain) suite au traitement.
Groupes biologiques analysés : WT (Sain), WT Treated (Sain traité), Disease (Malade), Rescue (Malade traité).

2. L'intelligence du fichier Metadata

Le script ne peut pas "deviner" le contenu des fichiers de comptage. Le fichier metadata.txt est le cerveau de l'analyse. Il permet de regrouper les échantillons par cohorte et par groupe biologique afin de définir précisément les contrastes statistiques pour l'extraction des DEG.

3. Nettoyage et "Matching" des échantillons

Les noms des échantillons dans les fichiers de données (ex: WT-1_S1) diffèrent souvent de ceux répertoriés dans les tableaux de suivi (ex: WT 1). Le script utilise une normalisation textuelle stricte :

# On retire les points, underscores, espaces et on passe en minuscule nom_propre = s.replace('.','').replace('_','').replace(' ','').lower()

Cela garantit que l'extraction des DEG repose sur les bons échantillons et qu'aucun individu n'est perdu à cause d'une faute de frappe.

4. Pourquoi normaliser en CPM ?

Avant d'extraire les DEG, il faut s'assurer que les variations observées ne sont pas dues à la technique. Si un échantillon a été séquencé avec 20 millions de lectures et un autre avec 40 millions, le second aura naturellement des chiffres deux fois plus hauts.

CPM = Lectures du gène Total des lectures de l'échantillon × 106

Cette étape rend les niveaux d'expression comparables entre toutes les cohortes, condition sine qua non pour un test statistique robuste.

5. La stratégie de Calibration (Le seuil de 1%)

Cette étape détermine quel groupe servira de référence pour définir la liste des DEG "Sain vs Malade". Le script automatise le choix pour s'adapter à la réponse biologique du traitement.

Le calcul du choix :

  1. Le script compare d'abord le groupe WT Treated au groupe WT.
  2. Il compte combien de gènes sont "déplacés" par le traitement seul (DEG avec $|Log2FC| > 1$ et $P < 0.05$).
  3. Il calcule le ratio : (Nb de gènes déplacés) / (Total gènes de la cohorte).

Divergence < 1%

Le traitement est considéré comme "neutre" sur le sujet sain. Le groupe WT Treated est utilisé comme référence.

Cela permet d'avoir un contrôle qui a reçu la même injection que le groupe Rescue.

Divergence ≥ 1%

Le traitement induit une réponse propre sur le sain. Le groupe WT pur est conservé comme référence.

On évite ainsi de "fausser" la ligne de base avec les effets secondaires du traitement.

Signification mathématique du Log2FC

Pour quantifier ces divergences (et identifier les DEG), nous utilisons le Log2 Fold Change. Le seuil de 1 garantit un changement d'amplitude biologique réelle :

Doublement (Ratio 2) : log2(2) = 1
Division par 2 (Ratio 0.5) : log2(0.5) = -1

L'utilisation de la base 2 rend les variations symétriques : une hausse d'un facteur 2 (+1) est visuellement comparable à une baisse d'un facteur 2 (-1).

6. Agrégation et Pseudocount

Pour chaque gène, nous calculons la moyenne d'expression par groupe biologique (WT, Disease, Rescue, etc.).

L'astuce du Pseudocount (+0.1)

Nous ajoutons 0.1 à chaque moyenne pour :

  • Éviter la division par zéro dans le calcul des ratios.
  • Stabiliser le Log2FC des gènes à très faible expression, évitant ainsi des variations artificielles dues au bruit technique.

7. Choix Statistique : Le T-test de Welch

Pour déterminer si un gène est significativement différent entre deux groupes, nous utilisons le Test $t$ de Welch.

t = X¯1-X¯2 s12n1 + s22n2
X̄₁ , X̄₂ Moyennes d'expression (CPM) des deux groupes.
s₁² , s₂² Variances respectives de chaque groupe.
n₁ , n₂ Nombre d'échantillons (réplicats) par groupe.
t Valeur calculée pour déterminer la p-value.

Pourquoi Welch plutôt que le test de Student classique ?

Le test de Student classique suppose l'homoscédasticité. Or, en biologie musculaire :

  • Hétérogénéité biologique : Les souris malades présentent souvent une variabilité plus forte.
  • Le test de Welch ajuste les degrés de liberté en fonction de la dispersion réelle de chaque groupe.

Note sur DESeq2 : Bien que standard, il nécessite les comptages bruts. Ce pipeline utilise Welch sur CPM pour rester polyvalent face à des sources hétérogènes (fichiers Excel déjà normalisés).

8. Extraction des DEG

Les gènes différentiellement exprimés sont isolés via deux filtres combinés :

Signature Patho (DEG)

Contraste [Disease vs Référence]. Filtre : |Log2FC| > 1 et P < 0.05.

Effet Rescue

Contraste [Rescue vs Disease]. Filtre : Significatif (P < 0.05) et direction opposée à la maladie.

Efficacité de la correction

Le pourcentage de correction est le rapport entre l'amplitude du traitement et l'amplitude de la maladie :

% Correction = (|Log2FC Rescue| / |Log2FC Patho|) × 100

9. Structure des Exports (Output)

Pour chaque cohorte (A à G), le pipeline génère 3 fichiers .csv distincts pour faciliter l'analyse en aval :

Genes_Analysis_Cohort_[].csv : Tous les gènes de la cohorte avec leurs calculs.
Genes_Signature_Patho_Cohort_[].csv : Uniquement la signature de la maladie.
Genes_Efficacite_Rescue_Cohort_[].csv : Uniquement les gènes corrigés par le traitement.
